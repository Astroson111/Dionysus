## 2. Firmware findings (source review)

**App map:** `.ino` (entry: homes servos → WiFi → Talk), **TalkApp** (voice round-trip, the heart), SettingsApp/SettingsStore (menu + NVS presets), **NetworkApp = no-op stub**, KaraokeApp (SD WAV + LRC), **GhostApp = unregistered stub**, **MenuApp = not included (legacy)**, CrescentMenu (drawer), AppManager/AppBase, ph3b3_face (violet face + moon-phase status tab), TouchKeyboard (on-device WiFi entry).

**Voice pipeline (TalkApp):** always-listening onset uses a **throwaway `monBuf[256]`** (`:123`) for RMS; onset fires after `rms > floor×6.0` for `ONSET_MIN_MS=150` (`:161-172`), then `M5.Mic.end()` + `_startRecording()` re-begins the mic (`:170,436-445`). The `PH_RECORDING` loop drains the mic **without yielding** (`:202-247`) — the documented DMA-overflow fix. TTS: manifest via `/chat/stream`, lazy `/tts/chunk/{sid}/{n}`, gapless double-buffered playback, dual watchdogs (`:823,854`).

**WiFi/provisioning:** blocking 12 s connect at boot (`:694`); `wifiSupervisorTick` retries slots every 15 s, `_syncNetworks` pulls `/stackchan/networks` after connect (`:236-271`); NVS `"sc"` keys ssid0/1/2, pass0/1/2, atomic srv_host/port/user/pass, wiped_v1. `Dio-Setup` SoftAP portal on no-creds or **16 consecutive failures (~4 min)** (`:485`); portal `while(true)` blocks until reboot.

**Servo/power — the crux:** `gentleHome` (`:632-640`) re-enables torque and trusts auto-angle-sync to have captured the servo's true physical angle; **torque is then never released** except on deliberate power-off (`safeHome`, `:619-625`). README landmine **f3767ae** (README:87-89) documents this exactly: *"The tilt (vertical) servo needs battery power to home — USB-only is current-limited and can't lift the head against gravity… This is power, not code; don't 'fix' it by firming up gentleHome() (more torque → more current → brown-out)."*

### Ranked firmware issues (most severe first)
1. **`loop()` fully blocked during a voice turn** (`:202-247`, `483-641`) — can't power-off mid-conversation; a WiFi drop isn't handled until the turn ends.
2. **Servo torque never disabled after boot homing** (`.ino:634`) — continuous hold current + blocking WiFi connect + LED ring = the brownout mechanism on battery.
3. **Servo jump-to-limit on boot** (`.ino:633-639`) — if the physical angle isn't captured (under-power on USB, or moved by hand while off), `moveY(450)` drives to a limit; `waitAxis` silently times out (5 s).
4. **First-word clipping** (`TalkApp.h:123,161-172`) — architectural: onset audio lives only in the throwaway `monBuf`, discarded before recording begins. Tuning can't fix it; a **pre-roll buffer** can (my attempt was the right idea, wrong power conditions).
5. **NetworkApp stub is live in the menu** (`.ino:716`) — tapping "Network" strands the user on a permanent CONNECTING face.
6–10. Karaoke lyric-clock drift (`KaraokeApp.h:328`) & ignores Volume (`:288`); ISRG cert bundle is dead code (all clients `setInsecure()`); `launchWifiPortal()` has no caller; GhostApp/MenuApp legacy carry.

## 3. Server-integration findings

**All 6 endpoints Dio calls + UDP telemetry are contract-compatible** with the current (Mnemosyne-enabled) server:

| Dio call | Server handler | Verdict |
|---|---|---|
| `POST /transcribe`, `/chat/stream`, `GET /tts/chunk/…`, `/health`, `/stackchan/networks`, UDP `7332` | all present, matching shapes | ✅ |

- **Auth:** HTTP Basic (NVS `srv_user/pass`) vs `.env` `AUTH_USER/PASS` — `X-Ph3b3-Device` is roster-only, not an auth factor. OK.
- **Mnemosyne:** Dio sends `X-Ph3b3-Device: stackchan`; auto-capture tags `source_device="stackchan"` and auto-recall is (correctly) not device-scoped → cross-device continuity works.

### Integration risks
1. **🐛 `dio` vs `stackchan` mismatch (my Mnemosyne code).** The Phase-2 "read story-state on boot" pattern queries `recent?device=dio&kind=state`, but Dio's data is tagged `stackchan` → returns nothing. Also **nothing writes `kind=state` for Dio** (DIO_STATE UDP is logged, never persisted). The Phase-2 boot-state feature is **not wired end-to-end**. Does NOT affect chat capture/recall. Fixable in server/memory_spine.
2. **Credential parity.** After the NVS wipe, the portal-entered server user/pass must match `.env` `AUTH_USER/PASS` exactly, or every Dio request 401s.
3. **First-run setup gate** would 303-redirect all device traffic to `/setup` until web-setup completes (one-time; not the current state).
